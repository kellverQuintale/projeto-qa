# Projeto QA - Portfólio de Testes de Software

## Sobre o projeto

Este projeto foi criado com o objetivo de demonstrar conhecimentos e práticas na área de Qualidade de Software (QA), incluindo planejamento de testes, criação de cenários, documentação de casos de teste e registro de defeitos.

O projeto simula atividades realizadas no dia a dia de um Analista de QA, seguindo boas práticas de testes de software.

---

## Objetivo

Garantir a qualidade das funcionalidades através da identificação de possíveis falhas, validação de regras de negócio e documentação dos resultados encontrados durante os testes.

---

## Atividades realizadas

✅ Análise de requisitos  
✅ Planejamento de testes  
✅ Criação de casos de teste  
✅ Testes funcionais  
✅ Testes de regressão  
✅ Testes exploratórios  
✅ Validação de cenários positivos e negativos  
✅ Documentação de bugs  
✅ Criação de cenários BDD utilizando Gherkin/Cucumber

## Tipos de testes aplicados

### Testes Funcionais
Validação das funcionalidades conforme requisitos definidos.

### Testes de Regressão
Garantia de que alterações não impactaram funcionalidades existentes.

### Testes Exploratórios
Análise livre do sistema buscando comportamentos inesperados e possíveis falhas.
